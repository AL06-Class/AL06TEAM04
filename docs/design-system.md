# 디자인 시스템

모든 화면이 WONDERDOGs 하나의 서비스처럼 보이도록 정리한 공통 디자인 시스템입니다.

## 목적

- 4명이 각자 화면을 나누어 작업해도 화면 톤과 컴포넌트 기준이 흔들리지 않게 합니다.
- 각 페이지가 임의의 색상, 여백, 버튼, 카드 스타일을 만들지 않게 합니다.
- 새 화면은 이 문서를 기준으로 만들고, 기준이 부족할 때만 문서를 먼저 수정합니다.
- 디자인 시스템은 개발을 막는 사전 승인 장치가 아니라, 화면의 결이 달라졌을 때 다시 맞추는 기준입니다.

## 기본값

### 디자인 방향

- WONDERDOGs는 경력단절 여성과 유연근무 기업을 연결하는 채용 플랫폼입니다.
- 화면은 채용 서비스의 신뢰감, 업무 도구의 실용성, 보라/라벤더 브랜드의 부드러움을 함께 가져갑니다.
- 장식보다 정보 구조와 사용성을 우선합니다.
- 모바일에서도 깨지지 않게 만들되, 데스크톱 작업 화면을 함께 고려합니다.
- 같은 역할의 UI는 같은 모양과 동작을 유지합니다.

### 유지해야 할 인상

- 밝고 깨끗한 채용 플랫폼
- 보라/라벤더 중심의 부드러운 브랜드 톤
- 기업 사용자가 빠르게 등록하고 관리할 수 있는 업무 도구 UI
- 구직자가 조건을 한눈에 이해할 수 있는 명확한 카드형 정보 구조
- 지도 기반 매칭 서비스처럼 보이는 위치 중심 화면

### 피해야 할 인상

- 화면마다 다른 컬러 테마 사용
- 진한 보라색을 너무 많이 사용해 무거워 보이는 화면
- 강한 그림자, 과한 그래픽, 복잡한 배경 장식
- 작은 글씨가 많아 보이는 관리자 도구 느낌
- 같은 기능인데 버튼, 배지, 카드 모양이 화면마다 다른 상태

### 색상 사용 원칙

- 기본 CTA는 `Purple 600`을 사용합니다.
- 선택된 상태는 보라색 테두리와 연한 라벤더 배경을 함께 사용합니다.
- 성공, 경고, 오류 색상은 상태 의미가 있을 때만 사용합니다.
- 배경은 흰색과 연한 라벤더를 중심으로 유지합니다.
- 페이지별로 별도 컬러 테마를 만들지 않습니다.

### 타이포그래피

- 기본 폰트는 시스템 폰트를 사용합니다.
- 별도 폰트 파일은 공유하지 않습니다.
- 과한 대형 제목이나 마케팅 페이지식 문구는 사용하지 않습니다.
- 버튼과 배지 안의 텍스트는 모바일에서도 잘리지 않게 합니다.

| 단계 | 권장 크기 | 굵기 | 사용 위치 |
| - | - | - | - |
| Display | 44~52px | 700~800 | 메인 페이지 큰 카피 |
| Page Title | 32~40px | 700 | 공고 관리, 과제 관리, 로그인 등 페이지 제목 |
| Section Title | 22~26px | 700 | 카드/폼 섹션 제목 |
| Card Title | 18~20px | 700 | 공고 카드, 과제 카드, 리스트 제목 |
| Body | 15~16px | 400~500 | 기본 본문, 설명 |
| Body Strong | 15~16px | 600~700 | 값, 주요 정보 |
| Caption | 12~14px | 400~500 | 날짜, 도움말, 메타 정보, 배지 |

### 문구 스타일

- 버튼 문구는 짧게 씁니다. 예: `다음 단계`, `등록 완료`, `지원하기`
- 안내 문구는 친절하되 길게 늘리지 않습니다.
- 상태 문구는 명확하게 씁니다. 예: `게시중`, `임시저장`, `마감`, `공고 연결됨`

| 화면 유형 | 레이아웃 기준 |

| 공고 등록 완료 | 중앙 완료 메시지 + 등록 요약 카드 + 다음 행동 버튼 |
| 과제 관리 | 탭 필터 + 3열 과제 카드 그리드 + 하단 등록 CTA |
| 과제 생성 | 좌측 조건 입력 + 우측 생성 후보 리스트 |
| 유연근무 공고 | 상단 검색/필터 + 좌측 지도 + 우측 추천 공고 리스트 |
| 공고 상세 | 좌측 공고 본문 + 우측 지도/사전과제 패널 + 하단 고정 CTA |

## 공통 컴포넌트

### Header / Navigation

- 좌측에는 WONDERDOGs 로고를 둡니다.
- 중앙에는 메뉴를 둡니다.
- 우측에는 알림, 로그인 버튼 또는 회원 드롭다운을 둡니다.
- 현재 메뉴는 보라색 텍스트와 하단 보라색 라인으로 표시합니다.

| 사용자 상태 | 메뉴 |
| - | - |
| 비로그인/일반 | 유연근무 공고, 경력자 찾기, 채용 도우미, 이용 안내, 고객센터 |
| 기업 회원 | 경력자 찾기, 공고 관리, 과제 관리, 지원자 관리, 유연근무 공고 |
| 구직자 회원 | 유연근무 공고, 이력서 관리, 지원 현황 |

### Button

| 유형 | 스타일 | 사용 예시 |
| - | - | - |
| Primary | 보라색 배경 + 흰색 텍스트 | `유연근무 공고`, `지원하기`, `다음 단계`, `등록 완료` |
| Secondary | 흰색 배경 + 보라색 border/text | `공고 보기`, `미리보기`, `이전`, `임시 저장` |
| Ghost | 배경 없음 + 보라색/회색 텍스트 | `전체 보기`, `필터 초기화`, `더보기` |
| Danger | 빨간 계열 border/text 또는 배경 | 삭제, 위험 작업 |
| Disabled | 회색 배경 + 회색 텍스트 | 선택 불가, 조건 미충족 |

| 크기 | 높이 | 사용 위치 |
| - | - | - |
| Large | 56~64px | 메인 CTA, 공고 등록 하단 CTA |
| Medium | 44~52px | 일반 버튼, 필터, 카드 CTA |
| Small | 32~40px | 테이블 액션, 배지 옆 버튼 |

- 한 화면의 가장 중요한 행동은 Primary 하나만 둡니다.
- 같은 줄에 Primary 버튼이 2개 이상 필요하면 우선순위를 다시 점검합니다.
- `임시 저장`은 Secondary로 유지합니다.
- `새 공고 등록하기`, `새 과제 등록하기`, `지원하기`는 Primary를 사용합니다.

### Card

| 유형 | 사용 위치 | 구성 |
| - | - | - |
| Summary Card | 대시보드, 등록 완료 | 아이콘/타이틀/핵심 값/CTA |
| List Card | 유연근무 공고, 과제 관리 | 상태 배지/타이틀/메타 정보/액션 |
| Detail Card | 공고 상세 | 섹션 제목/본문/리스트 |
| Preview Card | 공고 등록 우측 | 입력 내용 요약/수정 버튼 |
| Notice Card | 안내 메시지 | 아이콘/짧은 안내 문구 |

- 배경은 흰색을 사용합니다.
- 테두리는 `Gray 200`을 사용합니다.
- Radius는 16~20px을 사용합니다.
- Shadow는 거의 없거나 약하게 사용합니다.
- 내부 여백은 24~32px을 사용합니다.
- 카드 내부 보조 영역은 `Lavender 50` 배경을 사용할 수 있습니다.

### Form Field

- 기본 구성은 Label, Required 표시 `*`, Input/Select/Textarea, Help text 또는 오류 문구입니다.
- 기업이 편하게 등록할 수 있도록 기본값과 드롭다운을 우선 사용합니다.
- 필수 입력값은 label 옆에 `*` 표시를 붙입니다.
- 긴 설명 입력은 textarea로 분리합니다.
- 숫자, 시간, 요일은 직접 입력보다 선택 UI를 우선 사용합니다.

| 컴포넌트 | 사용 위치 | 기준 |
| - | - | - |
| Text Input | 공고명, 주소, 검색어 | 높이 44~52px, placeholder 명확히 |
| Select | 업종, 직무, 경력 수준, 급여 범위 | 드롭다운 우선, 직접 입력 최소화 |
| Textarea | 주요 업무, 자격 요건, 안내사항 | 글자 수 카운터 표시 |
| Chip Input | 필수 업무 스킬, 주요 업무 태그 | 선택된 값은 pill 형태로 표시 |
| Toggle | 시간 협의 가능 등 on/off | 선택 상태는 보라색 |
| Segmented Button | 근무 방식, 검색 기준 | 선택 항목은 보라색 border + 연한 배경 |

### Badge / Status

| 구분 | 상태값 | 표시명 | 스타일 |
| - | - | - | - |
| 공고 | `posted` | 게시중 | Green badge |
| 공고 | `draft` | 임시저장 | Gray 또는 Blue soft badge |
| 공고 | `closed` | 마감 | Gray badge |
| 공고 | `recommended` | 추천 | Purple soft badge |
| 과제 | `available` | 사용 가능 | Green badge |
| 과제 | `draft` | 임시저장 | Gray badge |
| 과제 | `linked` | 공고 연결됨 | Blue badge |
| 매칭 | `fullMatch` | 완전 매칭 | Green badge |
| 매칭 | `partialMatch` | 부분 매칭 | Orange badge |
| 매칭 | `dayPriority` | 요일 우선 | Orange soft badge |
| 매칭 | `noTask` | 과제 없음 | Gray badge |
| 매칭 | `hasTask` | 실무 과제 있음 | Blue badge |

- 배지는 한 줄에 3~5개를 넘기지 않습니다.
- 중요한 상태 배지는 카드 상단 또는 제목 옆에 둡니다.
- 보조 조건은 pill tag로 표시합니다.

### Tabs / Filter Bar

- 활성 탭은 보라색 텍스트와 보라색 하단 라인 또는 보라색 border로 표시합니다.
- 비활성 탭은 회색 텍스트로 표시합니다.
- 카운트는 연한 보라색 원형/캡슐 배지로 표시합니다.
- 필터는 한 줄 카드 안에 배치합니다.
- 필터 초기화는 Ghost 버튼으로 둡니다.
- 모바일에서는 필터를 2열 또는 접힘 영역으로 전환합니다.

예시:

- 공고 관리: `전체`, `게시중`, `임시저장`, `마감`
- 과제 관리: `전체`, `임시저장`, `사용 가능`, `공고 연결됨`

### Stepper

공고 등록 화면은 4단계 stepper를 사용합니다.

1. 회사 기본 정보
2. 유연근무 조건
3. 업무 상세
4. 과제

| 상태 | 표현 |
| - | - |
| 완료 | 체크 아이콘 + 연한 보라 배경 |
| 현재 | 보라색 원형 숫자 + 보라색 텍스트 |
| 예정 | 회색 원형 숫자 + 회색 텍스트 |

- 단계명은 위 4개로 통일합니다.
- 4단계 과제 화면의 상단 탭은 `과제 연결 안 함`, `기존 과제에서 선택`만 사용합니다.
- `새 과제 생성하기`는 탭으로 만들지 않고, 기존 과제 선택 영역 하단의 버튼으로 유지합니다.

### Table

- 공고 관리처럼 관리 목적의 목록은 테이블을 사용합니다.
- 기본 컬럼은 공고명, 직무, 근무 방식, 게시 상태, 최근 수정일, 관리 순서를 권장합니다.
- Header 배경은 아주 연한 회색 또는 흰색을 사용합니다.
- Row hover는 연한 라벤더 배경을 사용합니다.
- 관리 버튼은 `편집`, `미리보기`, `더보기` 순서로 둡니다.

### Map Panel

- 지도 화면은 실제 지도가 붙기 전에도 서비스 구조가 보이도록 지도 패널을 유지합니다.
- 지도는 좌측, 추천 리스트는 우측 배치를 기본으로 합니다.
- 위치 검색은 `현재 위치`, `지하철역`, `주소 검색` 탭/버튼으로 구분합니다.
- 매칭 안내 문구는 지도 위 또는 검색 영역 상단에 한 번만 노출합니다.

### Preview Panel

- 공고 등록 1~4단계의 우측에는 공고 미리보기 패널을 둡니다.
- 아직 입력되지 않은 섹션은 `입력 시 표시됩니다`로 표시합니다.
- 미리보기는 입력값을 확인하는 용도이며, 편집은 좌측 폼에서 합니다.
- 긴 내용은 접어서 `전체 보기`를 제공합니다.

## 반응형 기준

### 데스크톱

- 2열 레이아웃을 적극 사용합니다.
- 지도 + 리스트를 병렬 배치합니다.
- 공고 등록은 좌측 폼 + 우측 미리보기를 유지합니다.

### 태블릿

- 2열을 유지하되 폭이 좁으면 미리보기 패널을 아래로 이동합니다.
- 카드 그리드는 2열로 변경합니다.

### 모바일

- 모든 화면은 1열로 쌓습니다.
- Header 메뉴는 접힘 처리합니다.
- 지도는 리스트 위 또는 아래에 배치합니다.
- 하단 CTA는 sticky bar로 유지할 수 있습니다.
- 버튼 텍스트가 잘리지 않게 높이를 늘립니다.

## 협업 운영 기준

- 새 화면을 만들 때 이 문서의 컴포넌트 이름을 기준으로 대화합니다.
- 같은 역할의 UI를 새로 만들지 말고 기존 컴포넌트 기준을 먼저 확인합니다.
- 새 색상, 새 버튼 스타일, 새 카드 스타일을 임의로 추가하지 않습니다.
- 필요한 새 패턴이 생기면 먼저 `임시 화면 전용`으로 만들고, 반복되면 이 문서에 추가합니다.
- 데이터 이름 사전과 화면 문구가 충돌하면 데이터 이름 사전을 우선합니다.
- 컴포넌트 코드는 라이브로 공유하지 않고, 디자인 기준만 공유합니다.

### 역할 분담 예시

| 담당 | 권장 화면 |
| - | - |
| 담당자 A | 메인, 로그인, 공통 Header |
| 담당자 B | 기업 대시보드, 공고 관리 |
| 담당자 C | 공고 등록 1~4단계, 등록 완료 |
| 담당자 D | 과제 관리, 과제 생성, 유연근무 공고, 공고 상세 |

담당 화면은 바뀌어도 되지만, 공통 Header, Button, Card, Badge, Form Field 기준은 동일하게 유지합니다.

## 구현 전 점검 체크리스트

### 공통

- [ ] Header 메뉴명이 화면마다 다르지 않은가?
- [ ] Primary 버튼이 한 화면에 너무 많이 있지 않은가?
- [ ] 카드 radius, border, shadow가 유사한가?
- [ ] 상태 배지 색상이 의미와 맞는가?
- [ ] 보라색을 과하게 사용하지 않았는가?
- [ ] 작은 글씨가 모바일에서 잘리지 않는가?

### 공고 등록

- [ ] 4단계 stepper 명칭이 통일되었는가?
- [ ] 4단계 과제 탭에 `새 과제 생성하기` 탭이 없는가?
- [ ] `새 과제 생성하기`는 하단 버튼으로만 있는가?
- [ ] 과제 연결 안 함 상태에서도 등록 완료가 가능한가?
- [ ] 우측 미리보기 패널이 입력 단계별로 자연스럽게 갱신되는가?

### 유연근무 공고/공고 상세

- [ ] 위치 + 요일 + 시간 + 업무 우선 매칭 안내가 들어갔는가?
- [ ] 완전 매칭이 없을 때 일부 조건 매칭 추천 안내가 있는가?
- [ ] 지도와 리스트의 정보가 서로 충돌하지 않는가?
- [ ] 공고 상세 상단에 유연근무 핵심 조건이 먼저 보이는가?

## 최종 결정

- 디자인 방향: 신뢰감 있는 채용 플랫폼과 부드러운 보라/라벤더 브랜드를 결합한 업무 도구 UI
- 메인 컬러: `#6C2FF2`
- 보조 색상: 흰색, 밝은 라벤더, 옅은 회색, 진한 회색, 중간 회색
- 폰트: 시스템 폰트
- 제목 크기: Display, Page Title, Section Title, Card Title, Body, Caption 단계 사용
- 버튼 기준: Primary, Secondary, Ghost, Danger, Disabled로 구분
- 입력창 기준: 라벨, 필수 표시, 안내 문구, 오류 문구를 함께 관리
- 선호 요소: 둥근 카드, 밝은 라벤더 배경, 상태 배지, 지도 기반 매칭 패널, 친절한 안내 문구
- 비선호 요소: 진한 보라색 과다 사용, 강한 그림자, 복잡한 장식, 너무 작은 글씨
- 카드/표 기준: 카드는 반복 목록과 독립 정보 묶음에 사용하고, 표는 관리 목적 목록에 사용
- 역할별 화면 레이아웃: 같은 간격, 카드, 표, 제목 체계를 유지
- 반응형 기준: 모바일에서 텍스트와 버튼이 잘리지 않게 유지
- 공통 컴포넌트 운영: 컴포넌트 코드는 라이브로 공유하지 않고, 디자인 기준과 이름만 공유. 반복되는 패턴은 후속 리팩토링에서 공통으로 통합
